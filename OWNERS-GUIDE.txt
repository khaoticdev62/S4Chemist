WHAT THIS IS
This is a portable Sims 4 mod authoring helper. It creates project folders, XML/script scaffolds, packaging zips, and changelogs. It also includes a wizard that asks simple questions and gives modding guidance before you build.

HOW TO ASK BIG HERM TO USE IT
- "Create a new mod project"
- "Scaffold a career/trait/buff/interaction/event/achievement/aspiration/whim/club/holiday/loot_action/testset/relationship/skill/motive"
- "Run the wizard for <type>"
- "Validate/build my project"
- "Generate a changelog"

WHAT YOU NEED
- Windows PC with Python 3.10+ installed
- The Sims 4 installed on this PC for install/doctor checks
- `rich`, `questionary`, and `textual` packages for source runs (`pip install rich questionary textual`); the portable .exe needs nothing extra

SHORTCUT
SK="C:\Users\thecr\sims4-mod-cli\s4chemist_cli.py"
(or just run `python s4chemist_cli.py ...` from this folder)

COMMAND REFERENCE
- python "$SK" init MyMod
- python "$SK" new <project> <kind> <name>
- python "$SK" generate <type> <name> [--param key=value]
- python "$SK" wizard <type> [name] [--param key=value]
- python "$SK" validate [path] [--strict]
- python "$SK" build [path] [--release]
- python "$SK" package [path] [--out-dir <dir>]
- python "$SK" changelog [path]
- python "$SK" install [path] [--to-dir <dir>]   (or set S4_MODS_DIR)
- python "$SK" uninstall [path] [--to-dir <dir>]
- python "$SK" config <path> key=value [...]
- python "$SK" tune-ids <path> [--flavor]
- python "$SK" tui [path]   (full dashboard)
- python "$SK" doctor
- python "$SK" version

DASHBOARD (TUI)
- Run: python "$SK" tui [path]   (path defaults to the current folder)
- Four tabs:
  - Pipeline: live phase table plus detail for the selected phase
  - Create: guided mod creation form — type picker with all 18 kinds, name input,
    per-type parameter fields, inline "name is required" check, Create button
  - Files: lazy directory tree with syntax-highlighted preview
  - Log: output stream of every command you run
- Sidebar: project path box, Init Project, Validate/Build/Package/Changelog/Tune IDs/
  Install/Doctor buttons, Pipeline actions (Next Actions/Unlock Phase/Reset),
  a Create button that jumps to the Create tab, and a quick Generate form
- Status bar shows the project, current phase, and a progress bar
- Guided creation runs against the project path shown in the dashboard, not your
  terminal folder
- Keys: q quit, r refresh, Ctrl+P command palette
- Layout scales with window size (sidebar is 20% of width, 22-38 columns)

WIZARD FLOW
1. Open terminal in your folder
2. Run: python "$SK" wizard career NightShift
3. Answer the prompts; defaults are shown in brackets
4. It creates the project if needed, writes files, and prints Brain Advice, Dependencies, and Next Steps
5. Scripting it: when input is not a terminal, the wizard skips prompts and uses defaults plus
   any --param key=value overrides (pass the name as an argument or --param name=...)

SUPPORTED MOD TYPES
career, trait, buff, interaction, event, achievement,
xml_snippet, ts4script, package,
aspiration, whim, club, holiday,
loot_action, testset, relationship, skill, motive

WHAT THE OUTPUT MEANS
- [VERIFIED] = tool exercised the workflow successfully
- [LOCAL] = needs your PC-specific path or setting
- [OK] Validation — 0 issues = clean project
- Built — ...zip = packaged release file path (report includes archive size + file count)
- Panel borders are state-colored: red = fail, green = ok, yellow = local, blue = info
- Colors turn off when output is piped, when NO_COLOR is set, or with --no-color;
  S4_ASCII=1 forces ASCII glyphs

TROUBLESHOOTING
- "Not a valid project": run init or wizard in that folder first
- "Unknown kind": check spelling; use one of the supported types above
- "MISSING Sims 4 Documents": install Sims 4 or set S4_MODS_DIR to your Mods folder
- validate lists one actionable line per issue; --strict adds placeholder checks (docs/validation.md)
